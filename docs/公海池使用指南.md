客户公海池模块已实现，CRM 核心功能之一！

- ✅ 定时任务自动处理

## 🎯 公海池规则

### 掉入公海的原因

| 原因代码 | 说明 | 触发方式 |
|---------|------|---------|
| 1 | 超期未跟进 | 定时任务自动执行 |
| 2 | 主动释放 | 销售手动操作 |
| 3 | 离职交接 | 管理员操作 |
| 4 | 其他 | 手动操作 |

### 超期未跟进规则

**默认规则**：下次跟进时间 < 当前时间

**示例**：
- 销售 A 设置客户下次跟进时间为 2026-03-10
- 当前时间为 2026-03-14
- 超期 4 天未跟进
- 系统自动将客户掉入公海池

### 定时任务

| 任务 | 执行时间 | 说明 |
|------|---------|------|
| 自动掉入公海 | 每天凌晨 2:00 | 检查超期未跟进客户 |
| 生成周报 | 每周一早上 8:00 | 发送公海池周报（待实现） |

## 📝 API 接口

### 1. 分页查询公海池客户

**接口**：`GET /api/customer-pool/page`

**参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| current | Integer | 否 | 当前页码（默认 1） |
| size | Integer | 否 | 每页数量（默认 10） |

**权限**：`customer:pool:query`

**示例**：
```bash
curl "http://localhost:8080/api/customer-pool/page?current=1&size=20" \
  -H "Authorization: $TOKEN"
```

---

### 2. 查询公海池统计

**接口**：`GET /api/customer-pool/stats`

**权限**：`customer:pool:query`

**响应**：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "total": 50,        // 公海池总数
    "todayDrop": 5,     // 今日掉入
    "weekDrop": 20,     // 本周掉入
    "claimed": 100      // 已领取
  }
}
```

---

### 3. 领取公海池客户

**接口**：`POST /api/customer-pool/claim/{poolId}`

**权限**：`customer:pool:claim`

**说明**：销售领取公海池客户，自动变为该销售的客户

**示例**：
```bash
curl -X POST http://localhost:8080/api/customer-pool/claim/123456 \
  -H "Authorization: $TOKEN"
```

**响应**：
```json
{
  "code": 200,
  "message": "领取成功",
  "data": true
}
```

---

### 4. 分配公海池客户

**接口**：`POST /api/customer-pool/assign/{poolId}?userId={userId}`

**权限**：`customer:pool:assign`

**说明**：管理员分配公海池客户给指定销售

**示例**：
```bash
curl -X POST "http://localhost:8080/api/customer-pool/assign/123456?userId=100" \
  -H "Authorization: $ADMIN_TOKEN"
```

---

### 5. 手动释放客户到公海池

**接口**：`POST /api/customer-pool/drop/{customerId}`

**权限**：`customer:pool:drop`

**参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| reason | Integer | 否 | 掉入原因（默认 4） |
| remark | String | 否 | 备注 |

**示例**：
```bash
# 主动释放客户
curl -X POST "http://localhost:8080/api/customer-pool/drop/789?reason=2&remark=客户已无意向" \
  -H "Authorization: $TOKEN"

# 离职交接
curl -X POST "http://localhost:8080/api/customer-pool/drop/789?reason=3&remark=销售离职交接" \
  -H "Authorization: $ADMIN_TOKEN"
```

## 🔧 定时任务配置

### 自动掉入公海池

**执行时间**：每天凌晨 2:00

**Cron 表达式**：`0 0 2 * * ?`

**逻辑**：
1. 查询所有 `next_follow_time < NOW()` 的有效客户
2. 逐个掉入公海池
3. 清空客户负责人
4. 记录掉入原因和备注

**日志示例**：
```
2026-03-14 02:00:00 INFO  开始执行公海池定时任务：检查超期未跟进客户
2026-03-14 02:00:01 INFO  找到 5 个超期未跟进的客户
2026-03-14 02:00:02 INFO  客户 A 已掉入公海池
2026-03-14 02:00:02 INFO  客户 B 已掉入公海池
2026-03-14 02:00:03 INFO  公海池定时任务执行完成：成功 5 个，失败 0 个
```

### 生成周报

**执行时间**：每周一早上 8:00

**Cron 表达式**：`0 0 8 ? * MON`

**功能**：（待实现）
- 统计上周公海池数据

## 📊 数据流转

### 客户掉入公海池流程

```
客户（有负责人）
    ↓
超期未跟进 / 主动释放
    ↓
创建公海池记录
    ↓
清空客户负责人
    ↓
客户变为公海客户
```

### 领取公海池客户流程

```
公海池客户（状态=在池中）
    ↓
销售领取
    ↓
更新客户负责人
    ↓
更新公海池状态=已领取
    ↓
客户变为该销售的客户
```

## 🎯 使用场景

### 场景 1：销售忘记跟进客户

**问题**：销售 A 有 10 个客户，忘记跟进，导致客户流失

**解决方案**：
1. 系统每天凌晨 2 点检查超期客户
2. 超期客户自动掉入公海池
3. 其他销售可以领取
4. 提高客户转化率

---

### 场景 2：销售离职

**问题**：销售 B 离职，名下 50 个客户需要交接

**解决方案**：
1. 管理员将 50 个客户批量释放到公海池
2. 原因选择"离职交接"
3. 分配给接手销售
4. 或者直接分配给新销售

---

### 场景 3：销售主动放弃客户

**问题**：销售 C 发现某个客户无意向，想放弃

**解决方案**：
1. 销售手动释放客户到公海池
2. 原因选择"主动释放"
3. 填写备注"客户已无意向"
4. 其他销售可以领取

---

### 场景 4：抢公海池客户

**问题**：公海池有优质客户，多个销售想领取

**解决方案**：
1. 销售查看公海池列表
2. 找到意向客户
3. 点击"领取"按钮
4. 先到先得

## 📝 最佳实践

### 1. 设置合理的跟进周期

```java
// 建议：根据客户等级设置不同跟进周期
// VIP 客户：3 天
// 普通客户：7 天
// 潜在客户：15 天

customer.setNextFollowTime(LocalDateTime.now().plusDays(7));
```

### 2. 及时跟进即将超期的客户

```java
// 查询明天即将超期的客户
LambdaQueryWrapper<Customer> wrapper = new LambdaQueryWrapper<>();
wrapper.between(Customer::getNextFollowTime, 
    LocalDateTime.now(), 
    LocalDateTime.now().plusDays(1));
List<Customer> list = customerService.list(wrapper);
```

### 3. 记录详细的公海池备注

```bash
# 好的备注
curl -X POST "http://localhost:8080/api/customer-pool/drop/789?reason=2&remark=客户预算不足，暂时无意向，建议 3 个月后再联系"

# 不好的备注
curl -X POST "http://localhost:8080/api/customer-pool/drop/789?reason=2&remark=无意向"
```

### 4. 定期清理公海池

```sql
-- 查询超过 1 年无人领取的客户
SELECT * FROM crm_customer_pool 
WHERE status = 0 
  AND drop_time < DATE_SUB(NOW(), INTERVAL 1 YEAR);
```

## 🔧 故障排查

### 问题 1：定时任务不执行

**检查**：
1. 确认 `@EnableScheduling` 已启用
2. 确认 Spring 配置正确
3. 查看日志是否有错误
4. 检查服务器时间是否正确

### 问题 2：客户未自动掉入公海池

**检查**：
1. 确认 `next_follow_time` 字段有值
2. 确认客户状态为有效（status=1）
3. 确认客户有负责人（owner_id 不为空）
4. 查看定时任务日志

### 问题 3：领取失败

**检查**：
1. 确认公海池记录状态为"在池中"
2. 确认客户存在
3. 确认权限足够
4. 查看错误信息
