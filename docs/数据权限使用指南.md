## 🎯 数据权限规则

### 角色数据范围

| 角色 | 数据范围 | 说明 |
|------|---------|------|
| **超级管理员** | 全部数据 | 查看所有数据 |
| **销售经理** | 全部数据 | 查看团队所有数据 |
| **销售专员** | 仅本人 | 只能查看自己的数据 |

### 数据权限效果

**客户管理示例**：

```sql
-- 超级管理员/销售经理
SELECT * FROM crm_customer;  -- 无过滤

-- 销售专员（userId=100）
SELECT * FROM crm_customer WHERE owner_id = 100;  -- 自动添加过滤
```

## 🚀 快速开始

### 1. 添加数据权限注解

在 Service 查询方法上添加 `@DataPermission` 注解：

```java
@Service
public class CustomerServiceImpl extends ServiceImpl<CustomerMapper, Customer> implements CustomerService {

    /**
     * 查询客户列表（自动添加数据权限过滤）
     */
    @Override
    @DataPermission(value = DataType.CUSTOMER, field = "owner_id")
    public List<Customer> listAll() {
        return list();  // 自动添加 WHERE owner_id = userId
    }

    /**
     * 分页查询客户列表
     */
    @Override
    @DataPermission(value = DataType.CUSTOMER, field = "owner_id")
    public Page<Customer> page(Page<Customer> page) {
        return super.page(page);  // 自动添加数据权限过滤
    }
}
```

## 📝 注解详解

### @DataPermission

**位置**：`com.crm.system.common.annotation.DataPermission`

**参数**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| value | DataType | CUSTOMER | 数据类型 |
| field | String | "owner_id" | 权限字段名 |
| filter | boolean | true | 是否过滤 |

---

### DataType 枚举

```java
public enum DataType {
    CUSTOMER,      // 客户
    OPPORTUNITY,   // 商机
    CONTRACT,      // 合同
    FOLLOW_UP      // 跟进记录
}
```

### 使用示例

#### 1. 客户管理

```java
@Service
public class CustomerServiceImpl implements CustomerService {

    @DataPermission(value = DataType.CUSTOMER, field = "owner_id")
    @Override
    public List<Customer> listAll() {
        return list();
    }

    @DataPermission(value = DataType.CUSTOMER, field = "owner_id")
    @Override
    public Page<Customer> page(Page<Customer> page) {
        return super.page(page);
    }
}
```

#### 2. 商机管理

```java
@Service
public class OpportunityServiceImpl implements OpportunityService {

    @DataPermission(value = DataType.OPPORTUNITY, field = "owner_id")
    @Override
    public List<Opportunity> listByCustomer(Long customerId) {
        LambdaQueryWrapper<Opportunity> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(Opportunity::getCustomerId, customerId);
        return list(wrapper);  // 自动添加 AND owner_id = userId
    }

    @DataPermission(value = DataType.OPPORTUNITY, field = "owner_id")
    @Override
    public Page<Opportunity> page(Page<Opportunity> page) {
        return super.page(page);
    }
}
```

#### 3. 合同管理

```java
@Service
public class ContractServiceImpl implements ContractService {

    @DataPermission(value = DataType.CONTRACT, field = "owner_id")
    @Override
    public List<Contract> listByCustomer(Long customerId) {
        LambdaQueryWrapper<Contract> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(Contract::getCustomerId, customerId);
        return list(wrapper);
    }
}
```

#### 4. 跟进记录

```java
@Service
public class FollowUpServiceImpl implements FollowUpService {

    @DataPermission(value = DataType.FOLLOW_UP, field = "customer_id")
    @Override
    public List<FollowUp> listByCustomer(Long customerId) {
        // 跟进记录通过 customer_id 间接关联 owner_id
        LambdaQueryWrapper<FollowUp> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(FollowUp::getCustomerId, customerId);
        return list(wrapper);
    }
}
```

## 🔧 工作原理

### AOP + MyBatis 拦截器流程

```
查询方法调用
    ↓
@DataPermission 注解？
    ↓
是 → DataPermissionAspect 拦截
    ↓
获取当前用户 ID（UserContext）
    ↓
获取用户角色列表
    ↓
计算数据权限 SQL
    ↓
超级管理员/经理 → "1=1"（不过滤）
销售专员 → "owner_id = userId"
    ↓
设置到 DataPermissionContext
    ↓
执行查询方法
    ↓
MyBatis 拦截器捕获 SQL
    ↓
自动添加 WHERE 条件
    ↓
返回过滤后的结果
    ↓
清除上下文
```

## 📊 SQL 过滤示例

### 1. 超级管理员

```java
// 用户 ID=1，角色=super_admin
@DataPermission(value = DataType.CUSTOMER, field = "owner_id")
public List<Customer> listAll() {
    return list();
}

// 生成的 SQL
SELECT * FROM crm_customer WHERE 1=1;  -- 无过滤
```

### 2. 销售经理

```java
// 用户 ID=2，角色=sales_manager
@DataPermission(value = DataType.CUSTOMER, field = "owner_id")
public List<Customer> listAll() {
    return list();
}

// 生成的 SQL
SELECT * FROM crm_customer WHERE 1=1;  -- 无过滤
```

### 3. 销售专员

```java
// 用户 ID=100，角色=sales
@DataPermission(value = DataType.CUSTOMER, field = "owner_id")
public List<Customer> listAll() {
    return list();
}

// 生成的 SQL
SELECT * FROM crm_customer WHERE owner_id = 100;  -- 只查自己的
```

### 4. 已有 WHERE 条件

```java
// 用户 ID=100，销售专员
@DataPermission(value = DataType.CUSTOMER, field = "owner_id")
public List<Customer> search(String name) {
    LambdaQueryWrapper<Customer> wrapper = new LambdaQueryWrapper<>();
    wrapper.like(Customer::getCustomerName, name);
    return list(wrapper);
}

// 生成的 SQL
SELECT * FROM crm_customer 
WHERE customer_name LIKE '%测试%' 
  AND owner_id = 100;  -- 自动追加
```

## 🎯 最佳实践

### 1. 所有查询都添加数据权限

```java
// ✅ 推荐：所有查询方法都添加
@DataPermission(value = DataType.CUSTOMER, field = "owner_id")
public List<Customer> listAll() { }

@DataPermission(value = DataType.CUSTOMER, field = "owner_id")
public Page<Customer> page(Page<Customer> page) { }

@DataPermission(value = DataType.CUSTOMER, field = "owner_id")
public List<Customer> search(String keyword) { }

// ❌ 不推荐：部分查询不添加（可能泄露数据）
public List<Customer> listAll() { }  // 无权限控制
```

### 2. 明确指定字段名

```java
// ✅ 推荐：明确指定字段
@DataPermission(value = DataType.CUSTOMER, field = "owner_id")

// ❌ 不推荐：使用默认值（可能不匹配）
@DataPermission
```

### 3. 结合权限拦截器使用

```java
@RestController
@RequestMapping("/api/customers")
public class CustomerController {

    // 功能权限 + 数据权限
    @GetMapping
    @RequirePermission("customer:query")           // 功能权限
    @OperationLog(module = "客户管理", type = "查询")  // 操作日志
    public Result<List<Customer>> list() {
        // Service 层自动添加数据权限
        List<Customer> list = customerService.listAll();
        return Result.success(list);
    }
}
```

### 4. 数据权限测试

## 🔧 故障排查

### 问题 1：数据权限不生效

**检查**：
1. 确认 `@DataPermission` 注解在 Service 层方法上
2. 确认 AOP 切面已启用
3. 确认 MyBatis 拦截器已注册
4. 确认用户已登录（UserContext 有值）

### 问题 2：SQL 语法错误

**检查**：
1. 确认字段名正确（如 `owner_id`）
2. 确认表中有该字段
3. 检查 SQL 日志，查看生成的 SQL

### 问题 3：所有人都能看到全部数据

**检查**：
1. 确认用户角色配置正确
2. 确认 `DataPermissionCalculator` 逻辑正确
3. 检查用户是否有 super_admin 角色
