## 🎓 测试编写规范

本项目的测试遵循以下规范：

1. **命名规范**: `test[Method]_[Scenario]() `
2. **结构**: Arrange-Act-Assert (AAA) 模式
3. **断言**: 使用 JUnit 5 断言 + Mockito 验证
4. **注释**: 每个测试类包含作者和日期

示例：
```java
@Test
void testAddFollowUp_Success() {
    // Arrange（准备）
    when(followUpMapper.insert(any())).thenReturn(1);

    // Act（执行）
    boolean result = followUpService.addFollowUp(testFollowUp);

    // Assert（断言）
    assertTrue(result, "添加跟进记录应该成功");
    verify(followUpMapper, times(1)).insert(any());
}
```

## 💡 测试心得

1. **Mock 的使用**: 使用 Mockito 隔离依赖，只测试目标类
2. **边界条件**: 测试 null、空集合、极端值
3. **验证行为**: 不仅验证结果，还要验证调用了正确的方法
4. **可读性**: 测试名称要清晰表达测试意图
