## 📝 测试示例

### 1. 客户服务测试

**测试文件**：`CustomerServiceTest.java`

```java
@ExtendWith(MockitoExtension.class)
class CustomerServiceTest {

    @Mock
    private CustomerMapper customerMapper;

    @InjectMocks
    private CustomerServiceImpl customerService;

    @Test
    void testSave_Success() {
        // Arrange
        when(customerMapper.insert(any())).thenReturn(1);

        // Act
        boolean result = customerService.save(testCustomer);

        // Assert
        assertTrue(result);
        verify(customerMapper, times(1)).insert(any());
    }

    @Test
    void testGetById_Exists() {
        // Arrange
        when(customerMapper.selectById(1L)).thenReturn(testCustomer);

        // Act
        Customer result = customerService.getById(1L);

        // Assert
        assertNotNull(result);
        assertEquals("测试客户", result.getCustomerName());
    }
}
```

## 🎯 测试最佳实践

### 1. 测试命名规范

```java
// ✅ 推荐：清晰描述测试场景
@Test
void testSave_Success() { }

@Test
void testSave_Fail() { }

@Test
void testGetById_Exists() { }

@Test
void testGetById_NotExists() { }

// ❌ 不推荐：模糊的命名
@Test
void test1() { }

@Test
void testSave() { }
```

### 2. AAA 模式（Arrange-Act-Assert）

```java
@Test
void testExample() {
    // Arrange - 准备测试数据
    Customer customer = new Customer();
    when(customerMapper.selectById(1L)).thenReturn(customer);

    // Act - 执行被测试的方法
    Customer result = customerService.getById(1L);

    // Assert - 验证结果
    assertNotNull(result);
    assertEquals("测试客户", result.getCustomerName());
    verify(customerMapper, times(1)).selectById(1L);
}
```

### 3. 测试覆盖边界条件

```java
@Test
void testCalculateRate_ZeroTotal() {
    // 测试除数为 0 的情况
    Double rate = calculateRate(0, 50);
    assertEquals(0.0, rate);
}

@Test
void testCalculateRate_NullTotal() {
    // 测试 null 值
    Double rate = calculateRate(null, 50);
    assertEquals(0.0, rate);
}

@Test
void testCalculateRate_Normal() {
    // 测试正常情况
    Double rate = calculateRate(100, 50);
    assertEquals(50.0, rate);
}
```

### 4. 使用 Mock 隔离依赖

```java
@Mock
private CustomerMapper customerMapper;  // Mock 数据库依赖

@InjectMocks
private CustomerServiceImpl customerService;  // 被测试的服务

@Test
void testSave() {
    // Mock 返回值，不真正访问数据库
    when(customerMapper.insert(any())).thenReturn(1);
    
    // 测试业务逻辑
    boolean result = customerService.save(customer);
    
    // 验证调用
    verify(customerMapper, times(1)).insert(any());
}
```

## 🔧 常见问题

### 问题 1：测试失败 "MockitoException"

**解决**：
- 确保添加了 `@ExtendWith(MockitoExtension.class)`
- 确保使用 `@Mock` 和 `@InjectMocks` 注解

### 问题 2：空指针异常

**解决**：
- 检查 Mock 对象是否正确初始化
- 使用 `when().thenReturn()` 设置返回值

### 问题 3：验证失败 "Wanted but not invoked"

**解决**：
- 检查方法是否真的被调用
- 检查参数是否匹配（使用 `any()` 或具体值）
