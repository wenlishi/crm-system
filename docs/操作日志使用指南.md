- ✅ 自动记录操作日志（AOP 切面）

## 🚀 快速开始

### 1. 添加日志注解

在 Controller 方法上添加 `@OperationLog` 注解：

```java
@RestController
@RequestMapping("/api/customers")
public class CustomerController {

    /**
     * 新增客户
     */
    @PostMapping
    @RequirePermission("customer:add")
    @OperationLog(module = "客户管理", type = "新增", description = "新增客户")
    public Result<Boolean> save(@RequestBody Customer customer) {
        boolean success = customerService.save(customer);
        return success ? Result.success("添加成功", true) : Result.error("添加失败");
    }

    /**
     * 删除客户
     */
    @DeleteMapping("/{id}")
    @RequirePermission("customer:delete")
    @OperationLog(module = "客户管理", type = "删除", description = "删除客户")
    public Result<Boolean> delete(@PathVariable Long id) {
        boolean success = customerService.delete(id);
        return success ? Result.success("删除成功", true) : Result.error("删除失败");
    }
}
```

## 🔧 工作原理

### AOP 切面流程

```
请求 → Controller 方法
         ↓
   @OperationLog 注解？
         ↓
       是 → OperationLogAspect 拦截
         ↓
   1. 记录开始时间
         ↓
   2. 获取请求信息（URL/方法/IP/浏览器）
         ↓
   3. 获取注解信息（模块/类型/描述）
         ↓
   4. 获取用户信息（UserContext）
         ↓
   5. 记录请求参数
         ↓
   6. 执行目标方法
         ↓
   7. 成功 → 记录成功状态
         ↓
   8. 失败 → 记录错误信息
         ↓
   9. 计算执行时长
         ↓
   10. 异步保存日志
         ↓
   11. 返回结果
```

## 📊 日志数据结构

### 操作日志表 `sys_operation_log`

| 字段 | 类型 | 说明 |
|------|------|------|
| log_id | BIGINT | 日志 ID |
| user_id | BIGINT | 用户 ID |
| username | VARCHAR | 用户名 |
| module | VARCHAR | 操作模块 |
| type | VARCHAR | 操作类型 |
| description | VARCHAR | 操作描述 |
| method | VARCHAR | 请求方法（GET/POST/PUT/DELETE） |
| url | VARCHAR | 请求 URL |
| params | TEXT | 请求参数（JSON） |
| result | VARCHAR | 响应结果 |
| duration | BIGINT | 执行时长（毫秒） |
| ip | VARCHAR | IP 地址 |
| location | VARCHAR | 操作地点（待实现） |
| browser | VARCHAR | 浏览器 |
| os | VARCHAR | 操作系统 |
| status | TINYINT | 操作状态（0 失败 1 成功） |
| error_msg | TEXT | 错误信息 |
| create_time | DATETIME | 创建时间 |

## 🛠️ API 接口

### 1. 分页查询操作日志

**接口**：`GET /api/operation-logs/page`

**参数**：
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| module | String | 否 | 操作模块 |
| type | String | 否 | 操作类型 |
| userId | Long | 否 | 用户 ID |
| current | Integer | 否 | 当前页码（默认 1） |
| size | Integer | 否 | 每页数量（默认 10） |

**示例**：
```bash
# 查询所有日志
curl http://localhost:8080/api/operation-logs/page \
  -H "Authorization: $TOKEN"

curl "http://localhost:8080/api/operation-logs/page?module=客户管理" \
  -H "Authorization: $TOKEN"

# 查询某个用户的操作日志
curl "http://localhost:8080/api/operation-logs/page?userId=1" \
  -H "Authorization: $TOKEN"
```

## 🔧 高级用法

### 1. 自定义日志内容

可以在 Service 层手动补充日志信息：

```java
@Service
public class ContractServiceImpl implements ContractService {

    @Autowired
    private OperationLogService operationLogService;

    @Override
    @Transactional
    public void signContract(Contract contract) {
        // 业务逻辑...
        contractMapper.insert(contract);

        // 手动记录详细日志
        OperationLog log = new OperationLog();
        log.setUserId(UserContext.getUserId());
        log.setModule("合同管理");
        log.setType("签订");
        log.setDescription("签订合同：" + contract.getContractNo());
        log.setResult("合同金额：" + contract.getAmount());
        operationLogService.logOperation(log);
    }
}
```

### 2. 批量操作日志

```java
@OperationLog(module = "客户管理", type = "导入", description = "批量导入客户")
@PostMapping("/import")
public Result<Integer> importCustomers(@RequestParam MultipartFile file) {
    List<Customer> customers = importService.importFromFile(file);
    
    // 记录导入详情
    OperationLog log = new OperationLog();
    log.setModule("客户管理");
    log.setType("导入");
    log.setDescription("批量导入客户 " + customers.size() + " 条");
    operationLogService.logOperation(log);
    
    return Result.success(customers.size());
}
```

## 📊 日志分析

### 1. 操作频率分析

```sql
-- 查询每天的操作数量
SELECT DATE(create_time) as date, COUNT(*) as count
FROM sys_operation_log
GROUP BY DATE(create_time)
ORDER BY date DESC
LIMIT 30;
```

### 2. 模块热度分析

```sql
-- 查询各模块的操作数量
SELECT module, COUNT(*) as count
FROM sys_operation_log
GROUP BY module
ORDER BY count DESC;
```

### 3. 用户活跃度分析

```sql
-- 查询用户操作排名
SELECT username, COUNT(*) as count
FROM sys_operation_log
GROUP BY username
ORDER BY count DESC
LIMIT 10;
```

### 4. 异常操作分析

```sql
-- 查询失败的操作
SELECT username, module, type, description, error_msg, create_time
FROM sys_operation_log
WHERE status = 0
ORDER BY create_time DESC
LIMIT 50;
```

## 🛡️ 安全与隐私

### 1. 敏感信息脱敏

日志中不应包含敏感信息：

```java
// ❌ 不推荐：记录完整密码
@OperationLog(description = "用户登录，密码：123456")

// ✅ 推荐：不记录敏感信息
@OperationLog(description = "用户登录")
```

### 2. 日志访问控制

只有管理员可以查看操作日志：

```java
@GetMapping("/page")
@RequirePermission("system:log:query")  // 限制只有管理员可访问
public Result<Page<OperationLog>> page(...) {
    // ...
}
```

### 3. 日志保留策略

定期清理旧日志：

```sql
-- 删除 3 个月前的日志
DELETE FROM sys_operation_log
WHERE create_time < DATE_SUB(NOW(), INTERVAL 3 MONTH);
```
