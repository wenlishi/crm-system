## 数据库操作

### 为用户添加角色

```sql
-- 为用户 1 添加销售经理角色
INSERT INTO sys_user_role (user_id, role_id) VALUES (1, 2);
```

### 为角色添加权限

```sql
-- 为销售经理角色添加"客户新增"权限
INSERT INTO sys_role_permission (role_id, permission_id) VALUES (2, 23);
```

### 查询用户的所有权限

```sql
SELECT DISTINCT p.*
FROM sys_permission p
INNER JOIN sys_role_permission rp ON p.permission_id = rp.permission_id
INNER JOIN sys_user_role ur ON rp.role_id = ur.role_id
WHERE ur.user_id = 1 AND p.deleted = 0;
```

## 权限拦截器集成（待实现）

### JWT 拦截器中添加权限校验

```java
@Component
public class JwtInterceptor implements HandlerInterceptor {
    
    @Autowired
    private PermissionService permissionService;
    
    @Override
    public boolean preHandle(HttpServletRequest request, 
                            HttpServletResponse response, 
                            Object handler) {
        // 1. 解析 Token 获取用户 ID
        Long userId = getUserIdFromToken(token);
        
        // 2. 获取请求的权限编码
        String permissionCode = getPermissionCodeFromRequest(request);
        
        // 3. 查询用户权限
        List<Permission> permissions = permissionService.listByUserId(userId);
        
        // 4. 检查是否有权限
        boolean hasPermission = permissions.stream()
            .anyMatch(p -> p.getPermissionCode().equals(permissionCode));
        
        if (!hasPermission) {
            response.setStatus(403);
            return false;
        }
        
        return true;
    }
}
```

## 前端集成示例（Vue3）

```vue
<template>
  <el-menu>
    <menu-item 
      v-for="menu in permissionTree" 
      :key="menu.permissionId"
      :menu="menu" 
    />
  </el-menu>

<script setup>
import { ref, onMounted } from 'vue'
import axios from 'axios'

    headers: { Authorization: getToken() }
  })

})
</script>
```

## 角色说明

### 超级管理员（super_admin）
- 拥有所有 53 个权限
- 可以访问所有模块
- 可以进行所有操作

### 销售经理（sales_manager）
- 客户管理：全部权限
- 跟进记录：全部权限
- 商机管理：全部权限（含销售漏斗）
- 合同管理：全部权限（含合同统计）
- 统计报表：全部权限
- **无系统管理权限**

### 销售专员（sales）
- 客户管理：查询、新增、修改（无删除）
- 跟进记录：查询、新增、修改（无删除）
- 商机管理：查询、新增、修改（无删除）
- 合同管理：查询、新增、修改（无删除）
- 统计报表：仅数据统计
- **无系统管理、无删除权限**

## 最佳实践

### 1. 权限命名规范
```
模块：资源：操作
示例：
- customer:add      # 客户新增
- customer:edit     # 客户修改
- customer:delete   # 客户删除
- customer:query    # 客户查询
- system:user:add   # 系统管理 - 用户新增
```

### 2. 权限分配原则
- 最小权限原则：只分配必要的权限
- 角色分离：不同角色权限不重叠
- 定期审计：定期检查权限分配情况

### 3. 权限校验位置
- 前端：控制菜单和按钮显示
- 后端：接口级别权限校验
- 数据库：行级数据权限（可选）

## 常见问题

### Q: 如何添加新角色？
1. 在 `sys_role` 表插入新角色
2. 在 `sys_role_permission` 表分配权限
3. 重启项目或刷新缓存

### Q: 如何修改角色权限？
```sql
-- 删除权限
DELETE FROM sys_role_permission WHERE role_id = 2 AND permission_id = 23;

-- 添加权限
INSERT INTO sys_role_permission (role_id, permission_id) VALUES (2, 23);
```

### Q: 用户有多个角色，权限如何计算？
用户权限 = 所有角色权限的并集（去重）

### Q: 如何实现数据权限？
例如：销售只能看自己的客户，经理可以看全部。
- 在 SQL 中添加数据过滤条件
- 根据用户角色动态拼接 WHERE 条件
