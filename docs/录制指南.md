## 📋 准备工作

### 1. 环境检查

```bash
# 检查 CRM 系统是否运行
curl http://localhost:8080/api/swagger-ui.html

# 检查 jq 是否安装（用于格式化 JSON）
jq --version

# 如果没有安装 jq
sudo apt install jq    # Linux
brew install jq        # Mac
```

### 2. 启动 CRM 系统

```bash
cd /home/ubuntu/.openclaw/workspace/crm-system

# 方式 A：Docker 启动（推荐）
docker-compose up -d

# 方式 B：IDEA 启动
# 找到 SystemApplication.java，点击运行

# 等待系统启动完成（约 30 秒）
```

### 3. 准备录制软件

**推荐工具**：

| 工具 | 平台 | 特点 |
|------|------|------|
| **OBS Studio** | 全平台 | 免费开源，功能强大 |
| **QuickTime** | Mac | 系统自带，简单易用 |
| **Camtasia** | Win/Mac | 专业录屏，可编辑 |

## 🎥 录制方式

### 方式一：自动脚本录制（推荐）⭐

**步骤**：

1. **打开终端**，调整字体大小（建议 16-18 号）

2. **开始录制**（OBS 或其他工具）

3. **运行演示脚本**：
   ```bash
   cd /home/ubuntu/.openclaw/workspace/crm-system
   ./api-demo-script.sh
   ```

4. **等待脚本执行完成**（约 2-3 分钟）

5. **结束录制**

**优点**：
- ✅ 自动化执行，不易出错
- ✅ 输出格式统一
- ✅ 可重复录制

---

### 方式二：手动 curl 演示

**步骤**：

1. **准备 Token**：
   ```bash
   # 登录获取 Token
   curl -X POST http://localhost:8080/api/auth/login \
     -H "Content-Type: application/json" \
     -d '{"username":"admin","password":"admin123"}' | jq '.'
   
   # 保存 Token
   export TOKEN="eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
   ```

2. **逐个执行 API 调用**（参考 `api-demo-responses.json`）

3. **边执行边讲解**

**优点**：
- ✅ 更灵活，可随时解释
- ✅ 适合教学视频

---

### 方式三：Swagger UI 演示

**步骤**：

1. **访问 Swagger**：
   ```
   http://localhost:8080/api/swagger-ui.html
   ```

2. **授权 Token**：
   - 点击右上角 "Authorizations"
   - 输入 Token
   - 点击 Authorize

3. **逐个接口演示**：
   - 展开接口
   - 点击 "Try it out"
   - 填写参数
   - 点击 "Execute"
   - 查看响应

**优点**：
- ✅ 图形界面，更直观
- ✅ 适合非技术人员观看

### 第一部分：用户认证（1 分钟）

> 首先是用户认证模块。
> 
> 我们使用 POST 方法调用登录接口，传入用户名和密码。
> 
> 大家可以看到，返回了 JWT Token 和用户信息。
> 
> 这个 Token 有效期 24 小时，后续所有请求都需要携带它。
> 
> 【展示 Token 结构】

---

### 第二部分：客户管理（3 分钟）

> 接下来是客户管理模块，这是 CRM 的核心功能。
> 
> **创建客户**：
> 我们创建一个新客户，填入公司名称、联系人、电话等信息。
> 返回了客户 ID：101，创建成功。
> 
> **查询列表**：
> 使用分页查询，可以看到系统中有 25 个客户。
> 每页显示 10 条，支持自定义页码和页面大小。
> 
> **查看详情**：
> 这是客户 101 的完整信息，包括来源、等级、状态等。
> 
> **更新客户**：
> 现在我们把客户升级为 S 级，状态改为成交。
> 更新成功。
> 
> **删除客户**：
> 系统采用软删除，数据可以恢复。

---

### 第三部分：跟进管理（2 分钟）

> 第三部分是跟进管理。
> 
> **添加跟进**：
> 我们为客户添加一条跟进记录。
> 类型是电话，内容记录了沟通情况。
> 还可以设置下次跟进时间，系统会提醒。
> 
> **查询历史**：
> 这是该客户的所有跟进记录，按时间倒序排列。
> 可以看到之前的跟进历史和跟进人。

---

### 第四部分：数据统计（1 分钟）

> 最后是数据统计模块。
> 
> **客户统计**：
> 系统共有 156 个客户，本周新增 12 个。
> 这里按等级和状态分类统计。
> 还有近 7 天的趋势图。
