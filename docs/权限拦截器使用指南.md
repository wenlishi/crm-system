## 🚀 快速开始

### 1. 使用权限注解

在 Controller 方法上添加 `@RequirePermission` 注解：

```java
@RestController
@RequestMapping("/api/customers")
public class CustomerController {

    /**
     * 查询客户列表
     */
    @GetMapping
    @RequirePermission("customer:query")
    public Result<List<Customer>> list() {
        List<Customer> list = customerService.listAll();
        return Result.success(list);
    }

    /**
     * 新增客户
     */
    @PostMapping
    @RequirePermission("customer:add")
    public Result<Boolean> save(@RequestBody Customer customer) {
        boolean success = customerService.save(customer);
        return success ? Result.success("添加成功", true) : Result.error("添加失败");
    }

    /**
     * 删除客户
     */
    @DeleteMapping("/{id}")
    @RequirePermission("customer:delete")
    public Result<Boolean> delete(@PathVariable Long id) {
        boolean success = customerService.delete(id);
        return success ? Result.success("删除成功", true) : Result.error("删除失败");
    }
}
```

## 📝 权限注解详解

### @RequirePermission

**位置**：`com.crm.system.common.annotation.RequirePermission`

**参数**：

| 参数 | 类型 | 说明 | 示例 |
|------|------|------|------|
| value | String | 单个权限编码 | `@RequirePermission("customer:add")` |
| anyOf | String[] | 多个权限（满足任一） | `@RequirePermission(anyOf = {"customer:add", "customer:edit"})` |
| all | boolean | 是否需要所有权限 | `@RequirePermission(anyOf = {...}, all = true)` |

---

### 使用示例

#### 1. 单个权限

```java
@RequirePermission("customer:add")
@PostMapping
public Result<Boolean> save(@RequestBody Customer customer) {
    // 只有拥有 customer:add 权限的用户才能访问
}
```

#### 2. 多个权限（满足任一）

```java
@RequirePermission(anyOf = {"customer:add", "customer:edit"})
@PostMapping
public Result<Boolean> save(@RequestBody Customer customer) {
    // 拥有 customer:add 或 customer:edit 任一权限即可
}
```

#### 3. 多个权限（需要所有）

```java
@RequirePermission(anyOf = {"customer:add", "customer:edit"}, all = true)
@PostMapping
public Result<Boolean> save(@RequestBody Customer customer) {
    // 必须同时拥有 customer:add 和 customer:edit 权限
}
```

## 🔧 工作原理

### 拦截器流程

```
请求 → JwtInterceptor
         ↓
    1. OPTIONS 请求？→ 放行
         ↓
    2. 登录/注册接口？→ 放行
         ↓
    3. 检查 Token？→ 无 Token → 返回 401
         ↓
    4. 验证 Token？→ 无效 → 返回 401
         ↓
    5. 获取用户 ID → 存入 UserContext
         ↓
    6. 检查 @RequirePermission？
         ↓
       有注解 → 校验权限 → 无权限 → 返回 403
         ↓
       无注解 → 放行
         ↓
    7. 执行 Controller 方法
         ↓
    8. afterCompletion → 清除 UserContext
```

## 👥 用户上下文

### UserContext 工具类

**位置**：`com.crm.system.common.utils.UserContext`

**方法**：

```java
// 获取当前登录用户 ID
Long userId = UserContext.getUserId();

// 获取当前登录用户名
String username = UserContext.getUsername();

// 清除上下文（拦截器自动调用）
UserContext.clear();
```

### 在 Service 中使用

```java
@Service
public class CustomerServiceImpl implements CustomerService {

    @Override
    public Customer getById(Long id) {
        // 获取当前用户 ID
        Long currentUserId = UserContext.getUserId();
        
        // 可以记录操作人、过滤数据权限等
        log.info("用户 {} 正在查询客户 {}", currentUserId, id);
        
        return customerMapper.selectById(id);
    }
}
```

## 🔑 权限校验工具

### PermissionChecker

**位置**：`com.crm.system.common.utils.PermissionChecker`

**方法**：

```java
@Autowired
private PermissionChecker permissionChecker;

// 检查单个权限
boolean hasPermission = permissionChecker.hasPermission(userId, "customer:add");

// 检查任一权限
boolean hasAny = permissionChecker.hasAnyPermission(userId, "customer:add", "customer:edit");

// 检查所有权限
boolean hasAll = permissionChecker.hasAllPermissions(userId, "customer:add", "customer:edit");
```

超级管理员自动拥有所有权限，无需在数据库中分配！

// PermissionChecker.java
if (userId == 1L) {
    return true; // 超级管理员拥有所有权限
}

## 📊 错误响应

### 401 未授权

```json
{
  "code": 401,
  "message": "未授权，请先登录",
  "data": null
}
```

**触发条件**：
- 未提供 Token
- Token 无效
- Token 过期

---

### 403 禁止访问

```json
{
  "code": 403,
  "message": "没有权限执行此操作：customer:delete",
  "data": null
}
```

**触发条件**：
- Token 有效但无权限

## 📝 最佳实践

### 1. 所有接口都添加权限注解

```java
// ✅ 推荐：明确指定权限
@GetMapping("/{id}")
@RequirePermission("customer:query")
public Result<Customer> getById(@PathVariable Long id) {
    // ...
}

// ❌ 不推荐：不添加权限注解（任何人都可访问）
@GetMapping("/{id}")
public Result<Customer> getById(@PathVariable Long id) {
    // ...
}
```

### 2. 使用有意义的权限编码

```java
// ✅ 推荐：清晰的权限编码
@RequirePermission("contract:approve")

// ❌ 不推荐：模糊的权限编码
@RequirePermission("contract:1")
```

### 3. 在 Service 层获取当前用户

```java
@Service
public class OrderServiceImpl implements OrderService {
    
    @Override
    @Transactional
    public void createOrder(Order order) {
        // 获取当前用户 ID 作为订单创建人
        Long currentUserId = UserContext.getUserId();
        order.setCreatorId(currentUserId);
        
        // 业务逻辑...
        orderMapper.insert(order);
    }
}
```

### 4. 数据权限过滤

```java
@Service
public class CustomerServiceImpl implements CustomerService {
    
    @Override
    public List<Customer> listAll() {
        Long currentUserId = UserContext.getUserId();
        
        // 普通销售只能看自己的客户
        LambdaQueryWrapper<Customer> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(Customer::getOwnerId, currentUserId);
        
        return customerMapper.selectList(wrapper);
    }
}
```

## 🔧 故障排查

### 问题 1：权限注解不生效

**检查**：
1. 确认拦截器已配置（WebConfig）
2. 确认路径匹配（`/api/**`）
3. 确认注解在方法上（不是类上）
4. 确认权限编码正确

### 问题 2：总是返回 403

**检查**：
1. 确认用户有对应权限
2. 确认 Token 有效
3. 确认权限编码拼写正确
4. 检查数据库中 `sys_role_permission` 表是否有数据

### 问题 3：超级管理员也无权限

**检查**：
1. 确认用户 ID = 1
2. 确认 Token 中的 userId 解析正确
3. 检查 `PermissionChecker.hasPermission()` 方法
